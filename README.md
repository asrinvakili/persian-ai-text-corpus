# Persian AI-Text Corpus

A parallel Persian-language dataset for benchmarking AI-generated text detection. Each topic appears in four versions: one human-written and three AI-generated by different large language models.

## Why this corpus

Most AI-text-detection datasets are English- or Chinese-only. Persian (and the broader Perso-Arabic script family) is under-resourced for this task. This corpus provides a controlled benchmark for three research questions:

1. **Detection.** Can a classifier reliably distinguish Persian LLM output from human writing?
2. **Cross-generator analysis.** Which generator is hardest to detect?
3. **Generalization.** Does training on multiple generators transfer to an unseen one?

The **matched design** — every topic generated by every model — eliminates topic-as-confound when comparing generators.

## Composition

| Source | Records | Origin |
| --- | ---: | --- |
| Human (Persian Wikipedia, pre-2020 revisions) | 1,000 | Direct extraction with `revision_id` |
| DeepSeek v3.1 | 1,000 | AvalAI gateway |
| GPT-4o-mini | 1,000 | AvalAI gateway |
| Gemini 2.5 Flash-Lite | 1,000 | AvalAI gateway |
| **Total** | **4,000** | |

**Domain coverage.** Five domains, 200 topics each: theoretical sciences, engineering, medical sciences, humanities, history & biography.

**Length distribution.**
- Human: mean 74.7 words (σ=32.7), range 40–251.
- AI: mean 66.8 words (σ=29.1), range 22–233.

**Default splits.** Train 70% / val 10% / test 20% — stratified by (label, category, model).

See [`docs/datasheet.md`](docs/datasheet.md) for the full datasheet.

## Schema

Each line of every JSONL file is a record with the following fields.

### Human record

```json
{
  "id": "0001",
  "human_id": "0001",
  "label": 0,
  "source": "human",
  "category": "علوم نظری",
  "subcategory": "نابرابری‌ها",
  "title": "نابرابری مارکوف",
  "num_words": 62,
  "revision_id": 27800612,
  "revision_date": "2019-12-13T05:18:36Z",
  "url": "https://fa.wikipedia.org/w/index.php?oldid=27800612",
  "text": "..."
}
```

### AI record

```json
{
  "id": "ai_0001",
  "human_id": "0001",
  "label": 1,
  "source": "ai",
  "model": "deepseek-v3.1",
  "category": "علوم نظری",
  "subcategory": "نابرابری‌ها",
  "title": "نابرابری مارکوف",
  "num_words": 56,
  "target_words": 62,
  "generation_date": "2026-06-17",
  "text": "..."
}
```

The `human_id` field is the join key. Each human record has exactly three matching AI records sharing the same `human_id`.

## Quick start

### Load with the standard library

```python
import json

def load_jsonl(path):
    with open(path, encoding="utf-8") as f:
        return [json.loads(line) for line in f if line.strip()]

train = load_jsonl("data/splits/train.jsonl")
print(f"{len(train)} records, fields: {list(train[0].keys())}")
```

### Load with Hugging Face Datasets

```python
from datasets import load_dataset

ds = load_dataset(
    "json",
    data_files={
        "train": "data/splits/train.jsonl",
        "validation": "data/splits/val.jsonl",
        "test": "data/splits/test.jsonl",
    },
)
print(ds)
```

### Pair human with AI (for matched analysis)

```python
import json
from collections import defaultdict

humans = {r["human_id"]: r for r in load_jsonl("data/full/human_texts.jsonl")}
ai_by_hid = defaultdict(list)
for r in load_jsonl("data/full/ai_texts.jsonl"):
    ai_by_hid[r["human_id"]].append(r)

# Every human has exactly 3 matched AI versions
hid = "0001"
print(humans[hid]["text"])
for ai in ai_by_hid[hid]:
    print(f"[{ai['model']}] {ai['text']}")
```

A complete loader script is in [`examples/load_dataset.py`](examples/load_dataset.py).

## Class imbalance — read before training

The matched design produces a 1:3 class ratio (1,000 human vs 3,000 AI). For training a balanced detector, choose one strategy:

**Option A — class weighting (recommended).**
Weight the human class 3× higher. Keeps all data, learns from every example.

```python
# PyTorch
import torch.nn as nn
criterion = nn.CrossEntropyLoss(weight=torch.tensor([3.0, 1.0]))

# scikit-learn
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(class_weight={0: 3.0, 1: 1.0})
```

**Option B — subsample to 1:1.**
For each `human_id`, randomly keep one of the three AI versions. Result: 1,000 + 1,000 = 2,000 records.

```python
import random
rng = random.Random(42)
balanced_ai = [rng.choice(ai_by_hid[hid]) for hid in humans]
```

For **cross-generator analysis** (which model is hardest to detect), use the full 3,000-AI corpus.

## Suggested experiments

- **Baseline detection.** Fine-tune ParsBERT on `train.jsonl`, evaluate on `test.jsonl`.
- **Ablation across generators.** Train on one model only; test on each held-out model.
- **Robustness to paraphrasing.** Apply back-translation (e.g., `persiannlp/mt5-small-parsinlu`) to test set and re-evaluate.
- **Per-domain accuracy.** Group `test.jsonl` by `category`; see where the detector fails.

## Limitations

- **Domain.** Encyclopedic prose only; informal and dialectal Persian are absent.
- **Generator coverage.** Three frontier LLMs accessed via AvalAI in June 2026. Models like Claude, Llama, and Persian-native LLMs are not included.
- **Time.** Generators evolve; this snapshot captures the June 2026 state.
- **Imbalance.** Inherent to the matched design (see above).
- **Medical-domain diversity.** Slightly lower subcategory diversity than other domains (669 candidate articles vs ~1,200 elsewhere).
